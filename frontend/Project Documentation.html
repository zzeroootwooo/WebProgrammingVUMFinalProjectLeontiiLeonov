Paradise Hotel Frontend Documentation

1. Project Overview

Paradise Hotel Frontend is a React application built for a hotel reservation practical project. The frontend consumes a provided backend API and presents features for guests and administrators. The application supports authentication, room search, reservation management, and an admin area.

2. Technologies Used

  • React
  • Vite
  • Axios
  • React Router
  • Framer Motion
  • Custom CSS

3. How to Run the Frontend

Before running the frontend, make sure the backend server is also started.

  1. Open a terminal and go to the frontend folder.
  2. Install dependencies with npm install.
  3. Start the development server with npm run dev.
  4. Open the local address shown by Vite, usually http://localhost:5173.

If needed, the frontend can use a custom backend URL through the environment variable VITE_API_BASE_URL.

4. Available Scripts

  • npm run dev - starts the development server
  • npm run build - builds the production version
  • npm run lint - runs ESLint checks
  • npm run preview - previews the production build

5. Project Structure

  • src/pages - page-level screens such as Home, Search, Login, Register, My Reservations, and Admin pages
  • src/components - shared components such as Navbar and ProtectedRoute
  • src/components/ui - reusable UI components such as Button, Card, Input, Modal, Alert, and others
  • src/context - authentication state management
  • src/services - API communication layer using Axios
  • src/layouts - main page layout
  • src/assets - static images and resources

6. Main Features

  • User registration
  • User login and logout
  • Protected routes for users and administrators
  • Search for available rooms by dates and guest count
  • Filtering by hotel name, city, rating, free parking, and wellness center
  • Create a reservation for an available room
  • Cancel a reservation
  • View personal reservations
  • Administration dashboard
  • Administration reservations page
  • Administration locations and rooms management

7. Main Pages and Routes

  • / - Home page
  • /about - About page
  • /login - Login page
  • /register - Register page
  • /search - Search available rooms
  • /my-reservations - User reservations page
  • /admin/dashboard - Admin dashboard
  • /admin/reservations - Admin reservations page
  • /admin/locations - Admin locations and rooms management

8. API Integration

Axios is used for all HTTP communication. A shared Axios instance is configured in src/services/api.js. Service files are separated by responsibility:

  • authService.js - authentication requests
  • locationService.js - public location and room search requests
  • reservationService.js - reservation creation and cancellation
  • adminService.js - admin-related requests

9. Validation and Error Handling

The frontend includes validation for login, registration, search forms, and admin forms. API errors are shown to the user through alert messages. Client-side route protection prevents unauthorized access to private and admin pages.

11. Final Notes

The frontend is designed to follow the provided backend contract without changing API DTOs. The project structure is organized for clarity and reuse, with separate folders for pages, shared components, UI components, services, and context.
